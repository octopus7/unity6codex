# UI Toolkit Hands-On Lab

This lab is isolated from existing game code and lives only under:

`Assets/Labs/UiToolkitHandsOn`

## Goal

Practice UI Toolkit step by step with a dedicated runtime lab:

1. Query and click event
2. Value changed callback and live preview
3. Runtime list generation and filtering

## Folder Layout

- `Runtime/`: lab runtime scripts
- `Editor/`: scene bootstrap menu
- `Resources/UiToolkitHandsOn/`: UXML/USS templates
- `Scenes/`: practice scene (generated by menu)

## Quick Start

1. Open Unity Editor.
2. Click `Tools > UI Toolkit Lab > Create Practice Scene (Safe)`.
3. Enter Play mode.
4. Use `Previous`, `Reset Step`, `Next` buttons to move through lessons.

If the scene already exists, the safe menu opens it without destructive reset.

## Practice Flow

### Step 1 - Query and Click Event

- Learn `Q<T>("name")` query and button callback.
- Edit `Lesson01_QueryAndClick.uxml` and try adding one extra control.

### Step 2 - Value Changed Callback

- Connect `TextField`, `SliderInt`, `ProgressBar`, `Label`.
- Edit `Lesson02_ValueChanged.uxml` and add one more field to preview.

### Step 3 - Runtime List + Filter

- Build button list into `ScrollView` at runtime.
- Filter items via `TextField` callback.

## Independence Notes

- No existing gameplay scripts are modified.
- No existing scene is overwritten by default.
- All learning assets are self-contained in this lab path.
